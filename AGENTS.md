# AGENTS.md

Правила для будущих Codex-итераций в этом репозитории.

- Работать маленькими PR и писать отчёты на русском языке.
- Каждую задачу брать из `docs/agent-tasks/PR-XXX.md`, если такой файл существует.
- После выполнения PR обязательно создать или обновить `docs/agent-reports/PR-XXX.md`.
- Отчёт писать на русском языке.
- В отчёте честно указывать ошибки, ограничения и непроверенные сценарии.
- Не писать в чат длинный отчёт, если отчёт уже сохранён в файле; достаточно короткой ссылки на файл отчёта.
- PR открывать сразу Ready for review, если проверки проходят; Draft не использовать по умолчанию.
- Не мержить PR без подтверждения пользователя.
- Если задача меняет UX, обязательно добавлять ручные сценарии проверки.
- Не добавлять backend, серверные функции, авторизацию, базу данных или внешние API.
- Не превращать проект в генератор гаданий или случайных раскладов.
- Держать сайт статическим, быстрым и удобным на телефоне во время живого расклада.
- Поддерживать композиционную модель: базовая карта + позиция + тип вопроса + ориентация + точечные overrides.
- Не смешивать схему расклада и тип вопроса: расклад отвечает за позиции, тип вопроса отвечает за смысловую линзу.
- Не дублировать полные значения всех карт для каждой позиции вручную.
- Не использовать в интерфейсе слово «шпаргалка».
- Проверять отсутствие горизонтальной прокрутки на мобильной ширине.
- Перед PR выполнять `npm run check`, `npm test`, `npm run build` и `git diff --check`.
