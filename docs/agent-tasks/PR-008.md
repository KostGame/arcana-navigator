# PR-008: Add structured short meanings for readings

## 1. Контекст

Проект: `KostGame/arcana-navigator`.

После PR-006 и PR-007 мобильный UX режима «Собрать расклад» стал компактнее. Осталась смысловая проблема: блок «Суть по открытым картам» и краткие карточки выбранных карт пока строятся эвристически из существующих частей чтения и глаголов.

Из-за этого некоторые формулировки могут звучать слишком общо или шаблонно. Нужно добавить отдельные короткие смыслы, написанные специально для быстрого устного чтения.

Задача взята из issue #11: https://github.com/KostGame/arcana-navigator/issues/11

## 2. Цель

Добавить структурированное поле `shortMeaning`, чтобы:

- «Суть» не бралась из обрезанной первой фразы;
- карточки выбранных карт звучали точнее;
- блок «Суть по открытым картам» стал более живым;
- пользователь мог быстро понять, что сказать по каждой карте и по раскладу в целом.

## 3. Ветка и PR

- Ветка: `feature/pr008-structured-short-meanings`
- PR: `PR-008: Add structured short meanings for readings`
- Статус PR: Draft

## 4. Что сделать

1. Добавить короткий смысл в модель данных, предпочтительно простое поле `shortMeaning: string`.
2. Минимально заполнить `shortMeaning` для:
   - 4 мастей;
   - Туз → 10;
   - 4 придворных карт;
   - 22 старших арканов.
3. Использовать `shortMeaning` в:
   - быстром разборе;
   - карточках выбранных карт в режиме «Собрать расклад»;
   - блоке «Суть по открытым картам»;
   - общей фразе «Что сказать».
4. Сохранить полные `summary`, внимание, анти-трактовку и `parts` в «Подробнее».
5. Улучшить синтез «Что сказать», чтобы он опирался на короткие смыслы выбранных карт, а не на техническую склейку `parts`.

## 5. Что не делать

- Не добавлять backend.
- Не добавлять внешние API.
- Не добавлять генератор случайных карт.
- Не добавлять историю раскладов.
- Не добавлять изображения карт.
- Не делать большой редизайн.
- Не переписывать все длинные значения карт.
- Не усложнять data model сверх необходимости.

## 6. Тесты и проверки

Добавить или обновить тесты:

1. `composeReading` или новый helper возвращает `shortMeaning`.
2. 10 Кубков имеет короткий смысл про эмоциональную полноту / свой круг.
3. Дьявол имеет короткий смысл про желание / привязку / сильный стимул.
4. Паж Пентаклей имеет короткий смысл про практический шаг / навык / обучение.
5. Summary собранного расклада использует `shortMeaning`, а не обрезанную первую фразу.
6. Быстрый режим не ломается.

Запустить:

- `npm run check`
- `npm test`
- `npm run build`
- `git diff --check`

## 7. Ручная проверка

На ширине 390px проверить:

- в выбранных картах виден короткий смысл без шаблонной рыбы;
- «Суть по открытым картам» стала читаться естественнее;
- фраза «Что сказать» помогает начать устный разбор;
- подробности остаются доступны через «Подробнее»;
- горизонтальной прокрутки нет.
