# PR-013: Improve reference add position and filters UX

## 1. Контекст

PR-012 уже смержен. В нём справочник карт получил действие `+ В расклад`, которое добавляет карту в активную или следующую пустую позицию. Перед дальнейшими итерациями нужно довести UX справочника: дать выбор конкретной позиции и расширить быстрые фильтры.

## 2. Цель

Сделать справочник удобнее как рабочий каталог для набора расклада:

- карта добавляется в следующую доступную позицию по умолчанию;
- пользователь может выбрать конкретную позицию расклада прямо на карточке;
- фильтры покрывают типы карт, масти, достоинства и придворные роли;
- карточки справочника становятся более компактными и каталожными.
- в режиме «Собрать расклад» выбранные карты доступны в липкой строке наверху страницы.

## 3. Ветка и PR

- Ветка: `feature/pr013-reference-position-filters-ux`
- PR: `PR-013: Improve reference add position and filters UX`
- PR открыть обычным PR без Draft.

## 4. Что сделать

- Заменить простую кнопку `+ В расклад` на компактное действие `＋ [позиция]`.
- В списке позиций первым вариантом оставить `Следующая`.
- Если выбрана конкретная пустая позиция, добавить карту туда.
- Если выбрана заполненная позиция, не перезаписывать карту и показать предупреждение: `Позиция уже заполнена. Измените её в раскладе или очистите позицию.`
- Не делать принудительную замену карты.
- Расширить фильтры справочника:
  - тип: `Все`, `★` Старшие, `◆` Младшие, `♛` Двор;
  - масть: Жезлы, Кубки, Мечи, Пентакли;
  - ранг: Туз, 2-10, Паж, Рыцарь, Королева, Король.
- Сделать карточки справочника компактнее: название, действие, краткая суть, глаголы, `Подробнее`.
- У пиктограмм оставить `aria-label` и `title`.
- Добавить в режим «Собрать расклад» sticky-строку выбранных карт/позиций наверху страницы.
- Тап по заполненной карте в sticky-строке открывает эту позицию в режиме изменения, чтобы выбрать карту заново.
- Тап по пустой позиции в sticky-строке делает её активной.

## 5. Что не делать

- Не добавлять `localStorage`.
- Не добавлять избранное.
- Не добавлять генератор карт.
- Не делать большой редизайн.
- Не перезаписывать заполненные позиции без отдельного явного действия.
- Не добавлять backend, внешние API, аккаунты или историю раскладов.

## 6. Тесты и проверки

- Добавление из справочника в `next available` работает как раньше.
- Добавление в конкретную пустую позицию работает.
- Попытка добавить в заполненную конкретную позицию не перезаписывает карту.
- Фильтр по рангу `8` возвращает `8 Кубков`.
- Фильтр по двору `Паж` возвращает `Паж Пентаклей`.
- Фильтр по типу `Старшие` возвращает Луну.
- Быстрый режим и обычный session picker не ломаются.
- Sticky-строка расклада позволяет открыть заполненную карту на замену.

Запустить:

- `npm run check`
- `npm test`
- `npm run build`
- `git diff --check`

## 7. Ручная проверка

- На 390px открыть `Справочник карт`.
- Найти Луну и добавить её в следующую позицию.
- Найти 8 Кубков, выбрать конкретную пустую позицию и добавить.
- Попробовать добавить карту в заполненную позицию и увидеть предупреждение.
- Проверить фильтры `★`, `◆`, `♛`, масти, `8`, `Паж`.
- В режиме «Собрать расклад» заполнить 2-3 карты и проверить, что sticky-строка остаётся сверху при прокрутке.
- Тапнуть заполненную карту в sticky-строке и убедиться, что открывается замена карты.
- Убедиться, что карточки не создают горизонтальную прокрутку.
- Проверить, что быстрый режим и обычный picker сессии работают.

## 8. Критерии готовности

- Справочник поддерживает выбор позиции при добавлении карты.
- Заполненные позиции не перезаписываются.
- Фильтры расширены и остаются компактными.
- Карточки справочника стали менее текстовыми.
- Sticky-строка выбранных карт работает на мобильной ширине.
- Проверки проходят.
- PR открыт не как Draft.
