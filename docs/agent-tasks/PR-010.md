# PR-010: Mobile UX move reading first and add session sticky context

## 1. Контекст

После последних PR мобильный интерфейс Arcana Navigator стал компактнее, но сценарий живого чтения с телефона всё ещё можно ускорить.

Главная проблема: в быстром режиме результат не должен теряться ниже справочных блоков, а в режиме «Собрать расклад» пользователю важно всегда видеть активный контекст и быстро прыгать к чтению.

Задача взята из issue #15: https://github.com/KostGame/arcana-navigator/issues/15

## 2. Цель

Сделать мобильный сценарий быстрее для живого чтения с телефона:

- результат должен появляться сразу после выбора карты;
- справка не должна мешать основной фразе;
- в сессионном режиме должен быть компактный sticky-контекст активной позиции;
- подробности должны оставаться доступными, но не конкурировать с главным чтением.

## 3. Ветка и PR

- Ветка: `feature/pr010-mobile-reading-first-sticky-context`
- PR: `PR-010: Mobile UX move reading first and add session sticky context`
- Статус PR: Open / Ready for review

## 4. Что сделать

1. В быстром режиме изменить порядок блоков:
   - блок выбора карты;
   - блок результата;
   - справка ниже результата.
2. Оставить справку ниже результата и сделать её раскрывающейся через общий блок «Справочник карт».
3. В результате быстрого режима заменить верхнюю формулировку `Текущий выбор` на `Что сказать`.
4. Изменить порядок внутри результата:
   - фраза для чтения;
   - суть;
   - глаголы;
   - подробнее.
5. На мобильном сократить hero:
   - короткий заголовок;
   - короткое описание;
   - подробное пояснение в `details` с summary `Что это?`.
6. В режиме «Собрать расклад» добавить sticky mini-bar:
   - активная позиция;
   - выбранная карта или «карта не выбрана»;
   - кнопка `К чтению`.
7. Кнопка `К чтению` должна прокручивать к блоку результата / сути по открытым картам.
8. Не создавать горизонтальную прокрутку.

## 5. Что не делать

- Не добавлять backend.
- Не добавлять внешние API.
- Не добавлять случайную генерацию карт.
- Не добавлять историю раскладов.
- Не добавлять localStorage.
- Не добавлять изображения карт.
- Не делать большой редизайн.
- Не менять смысловые данные карт без необходимости.

## 6. Тесты и проверки

Добавить или обновить тесты, если меняется логика рендера или helpers.

Минимум проверить:

- быстрый режим не ломается;
- `shortMeaning` и фраза для чтения остаются в результате;
- session summary не ломается;
- кнопка / обработчик `К чтению` не ломает существующие события.

Запустить:

- `npm run check`
- `npm test`
- `npm run build`
- `git diff --check`

## 7. Ручная проверка

На ширине 390px проверить:

- в быстром режиме результат виден без долгой прокрутки;
- справка находится ниже результата и не мешает;
- верх результата называется `Что сказать`;
- внутри результата первой идёт фраза для чтения, затем суть, глаголы, подробнее;
- hero стал короче;
- в режиме «Собрать расклад» виден sticky mini-bar;
- активная позиция понятна;
- кнопка `К чтению` прокручивает к результату;
- горизонтальной прокрутки нет;
- подробности не мешают главной фразе.
