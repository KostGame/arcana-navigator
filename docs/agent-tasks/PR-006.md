# PR-006: Compact mobile controls and real spread essence

## 1. Контекст

Проект: `KostGame/arcana-navigator`.

После PR-004 интерфейс стал компактнее, но ручная мобильная проверка показала две проблемы в режиме «Собрать расклад»:

- кнопки действий в карточках позиций всё ещё занимают слишком много места;
- текст выбранных карт местами звучит как повторяющаяся заготовка и не помогает быстро понять, что говорить.

Задача взята из issue #7: https://github.com/KostGame/arcana-navigator/issues/7

## 2. Цель

Сделать режим «Собрать расклад» удобнее для живого чтения с телефона:

- меньше крупных кнопок;
- больше сути по всем выбранным картам;
- сначала показывать, что сказать;
- детали держать ниже и в раскрывающихся блоках.

## 3. Ветка и PR

- Ветка: `feature/pr006-compact-mobile-controls-essence`
- PR: `PR-006: Compact mobile controls and real spread essence`
- Статус PR: Draft

## 4. Что сделать

1. Заменить крупные текстовые кнопки в карточках позиций на компактные icon-кнопки:
   - `Изменить` -> `✎`;
   - `Очистить` -> `×`;
   - добавить `aria-label="Изменить карту"` и `aria-label="Очистить позицию"`;
   - уменьшить ширину кнопок на мобильной ширине.
2. В режиме изменения оставить компактное подтверждение выбора и компактную отмену с доступным `aria-label`.
3. Убрать ощущение «рыбы» в тексте выбранных карт:
   - не выводить повторяющуюся шаблонную фразу в каждой карточке;
   - показывать реальный краткий смысл из данных карты, позиции и глаголов.
4. Добавить верхний блок «Суть по открытым картам» в результатах режима «Собрать расклад»:
   - краткая линия по заполненным позициям;
   - блок «Что сказать» с одной общей фразой;
   - блок должен быть важнее списка отдельных карт.
5. Упростить карточки выбранных карт до формата:
   - позиция и карта;
   - смысл;
   - глаголы;
   - подробности в закрытом `details`.
6. Проверить мобильную ширину около 390px:
   - кнопки маленькие;
   - нет горизонтальной прокрутки;
   - список выбранных карт не выглядит повторяющейся простынёй.

## 5. Что не делать

- Не добавлять backend.
- Не добавлять внешние API.
- Не добавлять генератор случайных раскладов.
- Не добавлять историю или хранение раскладов.
- Не добавлять изображения карт.
- Не делать большой редизайн.
- Не переписывать все данные карт.

## 7. Ручная проверка

Проверить на ширине около 390px:

- кнопки действий в позициях стали компактными;
- верхний блок даёт суть по открытым картам;
- карточки выбранных карт не повторяют шаблонную фразу;
- понятно, что сказать по раскладу;
- подробности доступны, но не мешают;
- горизонтальной прокрутки нет.
