# PR-012: Add cards from reference to current spread

## 1. Контекст

После PR-011 в проекте появился отдельный режим «Справочник карт». Теперь справочник должен стать рабочим инструментом для живого набора расклада: пользователь находит карту и добавляет её в текущий расклад без повторного выбора через inline picker.

Источник задачи: https://github.com/KostGame/arcana-navigator/issues/19

## 2. Цель

Добавить на карточки конкретных карт в справочнике действие «+ В расклад», которое добавляет карту в активную или следующую пустую позицию текущей сессии расклада.

## 3. Ветка и PR

- Ветка: `feature/pr012-add-cards-from-reference-to-current-spread`
- PR: `PR-012: Add cards from reference to current spread`
- PR открыть обычным PR без Draft.

## 4. Что сделать

- Добавить в `CardReferenceEntry` поле `selection` для старших арканов, младших карт и придворных карт.
- Не добавлять `selection` для карточек мастей и достоинств.
- На карточках карт в режиме «Справочник карт» показать компактную кнопку `+ В расклад`.
- При нажатии добавить карту в активную пустую позицию текущего расклада.
- Если активная позиция заполнена, добавить карту в следующую пустую позицию.
- Если все позиции заполнены, не перезаписывать карты и показать мягкое сообщение.
- После добавления показать компактный статус: `Добавлено: Карта → Позиция`.
- Добавить кнопку `К раскладу`, которая переключает интерфейс в режим «Собрать расклад».
- Ориентация карты из справочника по умолчанию: `upright`.
- Использовать существующую session-логику и не дублировать трактовки.

## 5. Что не делать

- Не добавлять backend.
- Не добавлять внешние API.
- Не добавлять генератор случайных карт.
- Не добавлять историю раскладов.
- Не добавлять `localStorage`.
- Не добавлять изображения карт.
- Не делать большой редизайн.
- Не перезаписывать заполненные позиции без явного действия пользователя.
- Избранное не входит в обязательный scope PR-012.

## 6. Тесты и проверки

- Reference entry для Луны содержит `selection` для major card.
- Reference entry для 8 Кубков содержит `selection` для minor card.
- Reference entry для Пажа Пентаклей содержит `selection` для court card.
- Карта из справочника попадает в активную пустую позицию.
- Если активная позиция заполнена, карта добавляется в следующую пустую позицию.
- Если все позиции заполнены, существующие карты не перезаписываются.
- Быстрый режим и обычный session picker не ломаются.

Запустить:

- `npm run check`
- `npm test`
- `npm run build`
- `git diff --check`

## 7. Ручная проверка

- Открыть «Справочник карт» на ширине около 390px.
- Найти Луну и нажать `+ В расклад`.
- Увидеть статус добавления.
- Перейти в «Собрать расклад» и увидеть Луну в позиции.
- Найти 8 Кубков и добавить следующей картой.
- Найти Пажа Пентаклей и добавить следующей картой.
- Убедиться, что заполненные позиции не перезаписываются случайно.
- Заполнить все позиции и проверить мягкое сообщение.
- Проверить отсутствие горизонтальной прокрутки.

## 8. Критерии готовности

- Справочник добавляет конкретные карты в текущий расклад.
- Служебные карточки мастей и достоинств не имеют кнопки добавления.
- Добавление не ломает защиту от перезаписи заполненных позиций.
- Статус добавления понятен на мобильной ширине.
- Проверки проходят.
- PR открыт не как Draft.
