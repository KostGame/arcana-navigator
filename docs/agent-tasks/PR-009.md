# PR-009: Update agent workflow to skip Draft PRs

## 1. Контекст

Пользователь изменил рабочий процесс: пока нужно пропускать Draft-стадию. Codex должен открывать обычный PR, потому что пользователь проверяет сразу финальную версию.

Текущие документы всё ещё требуют открывать PR как Draft и писать в отчётах «PR открыт как Draft». Это надо синхронизировать с новым процессом.

Задача взята из issue #13: https://github.com/KostGame/arcana-navigator/issues/13

## 2. Цель

Обновить правила агентского workflow так, чтобы будущие PR открывались сразу Ready for review, без Draft-статуса.

## 3. Ветка и PR

- Ветка: `feature/pr009-agent-workflow-ready-prs`
- PR: `PR-009: Update agent workflow to skip Draft PRs`
- Статус PR: Open / Ready for review

## 4. Что сделать

1. Обновить `AGENTS.md`:
   - заменить правило «PR открывать как Draft» на «PR открывать сразу Ready for review, если проверки проходят»;
   - оставить правило «не мержить PR без подтверждения пользователя»;
   - отчёт по-прежнему сохранять в `docs/agent-reports/PR-XXX.md`.
2. Обновить `docs/agent-tasks/README.md` и `docs/agent-tasks/TEMPLATE.md`:
   - убрать требование Draft;
   - указать, что PR открывается обычным PR;
   - если есть риск или непроверенный UI, это указывать в отчёте, но не использовать Draft по умолчанию.
3. Обновить `docs/agent-reports/README.md` и `docs/agent-reports/TEMPLATE.md`:
   - поле «Статус PR» должно допускать `Open / Ready for review`, а не только Draft;
   - добавить поле «Можно мержить: да/нет, при каких условиях».
4. Обновить `docs/ROADMAP.md` и `CHANGELOG.md`.
5. Опционально обновить `docs/agent-reports/PR-008.md`, если статус там не соответствует фактическому состоянию после merge.

## 5. Что не делать

- Не менять продуктовый UI.
- Не менять данные карт.
- Не добавлять backend.
- Не добавлять API.
- Не добавлять localStorage.
- Не добавлять генератор случайных карт.

## 6. Тесты и проверки

Запустить:

- `npm run check`
- `npm test`
- `npm run build`
- `git diff --check`

## 7. Ручная проверка

- Проверить, что в актуальных workflow-документах больше нет требования открывать PR как Draft по умолчанию.
- Проверить, что правило «не мержить без подтверждения пользователя» сохранено.
- Проверить, что формат отчёта содержит поле «Можно мержить».

## 8. Критерии готовности

- Файл задачи создан в `docs/agent-tasks/PR-009.md`.
- Workflow-документы обновлены под обычные PR без Draft.
- Отчёт создан в `docs/agent-reports/PR-009.md`.
- Проверки прошли.
- PR открыт не как Draft.

## 9. Формат отчёта

После выполнения в чат написать:

```text
PR-009 открыт не как Draft.
Отчёт сохранён в docs/agent-reports/PR-009.md.
PR: <ссылка>
Статус проверок: pass/fail
```
