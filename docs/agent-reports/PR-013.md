# Отчёт Codex по PR-013

## 1. Кратко

PR-013 доводит UX справочника после PR-012: карту можно добавить не только в следующую доступную позицию, но и в конкретную выбранную позицию расклада. Справочник получил компактные фильтры по типу, масти, рангу и двору, а текущий расклад теперь виден в sticky-строке сверху.

## 3. Что реализовано

- На карточках конкретных карт действие стало компактным: `＋` и select позиции.
- В select первым вариантом стоит `Следующая`; это сохраняет поведение PR-012.
- При выборе конкретной пустой позиции карта добавляется туда.
- При выборе заполненной позиции карта не перезаписывается, показывается предупреждение.
- Фильтры справочника сгруппированы: тип, масть, ранг.
- Добавлены фильтры `★`, `◆`, `♛`, масти, Туз-10, Паж, Рыцарь, Королева, Король.
- Карточки справочника стали каталожнее: название, действие, короткая суть, глаголы, `Подробнее`.
- В режимах «Справочник карт» и «Собрать расклад» добавлена sticky-строка текущих карт расклада.
- Тап по заполненной карте в sticky-строке открывает позицию в режиме изменения.
- Тап по пустой позиции в sticky-строке делает её активной.

## 8. Риски и ограничения

- Sticky-строка использует внутреннюю горизонтальную прокрутку для чипов на узком экране; горизонтальную прокрутку страницы это не должно создавать.
- Выбор конкретной позиции из справочника не заменяет заполненную карту: замену нужно делать через режим «Собрать расклад».
- Browser smoke-test был частично ограничен вводом текста в in-app browser, поэтому ключевые сценарии фильтрации дополнительно закреплены unit-тестами.
